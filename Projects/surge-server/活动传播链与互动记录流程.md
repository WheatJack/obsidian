## 一、整体流程

```
APP代理人分享 → H5客户打开 → 客户浏览/报名/签到/转发
      ↓              ↓                  ↓
  传播链接口      传播链接口         行为数据写入
  (首次分享)     (客户打开)        + 消息通知
```

**materialType = '1'** (MaterialConstant.MATERIAL_TYPE_ACTIVITY)

---

## 二、详细链路

### 2.1 APP 代理人分享活动

**触发入口**: 活动详情页 → 右上角分享按钮 / 邀请报名按钮

```
代理人在 App 点击"分享/邀请报名"
    ↓
App 调 trackingMaterial({
    materialId: 活动ID,
    materialType: '1',        ← 活动类型
    userId: 代理人ID,          ← 从 uni.getStorageSync('userInfo').id
    code: '',                  ← App 端为空
    chain: '',                 ← 首次分享为空
    senderId: '',
    directAgentId: '',
    parentPropagationId: '',
    shareTime: ''
})
    ↓
后端处理:
    1. userId 有值 → 查 blade_user → 找到 → isReaderAgent = true
    2. setCustomerType(1)    // 代理人
    3. checkMaterialExists → surgeActivityMapper.getActivityNameById → 活动名称
    4. 生成唯一 chain、browsingId、shareTime
    5. 记录代理人浏览行为
    6. fillActivityInfo() 填充活动标题+封面到消息JSON
    ↓
返回: {
    userId: 代理人ID,
    senderId: 代理人ID,
    directAgentId: 代理人ID,
    chain: 唯一链ID,
    shareTime: 时间戳,
    browsingId: 浏览记录ID,
    customerType: 1
}
    ↓
App 拼装分享链接:
    ${shareUrl}activity/?id=活动ID
        &browsingId=xxx
        &chain=xxx
        &directAgentId=代理人ID
        &senderId=代理人ID
        &shareTime=xxx
        &parentPropagationId=xxx
    ↓
弹出分享面板（微信/朋友圈/QQ/WhatsApp/复制/系统分享）
    ↓
分享封面：posterUrl + OSS缩略图处理(?x-oss-process=image/resize,w_200)
分享标题：活动名称
分享摘要：邀请您参加活动：活动名称
```

### 2.2 微信客户打开活动链接

```
客户在微信点开链接
    ↓
initHandle():
    1. initWxConfig() → 初始化微信 JS-SDK
    2. 检查 localStorage.USERID → 没有
    3. 检查 userStore.wxUserInfo → 没有
    4. 调 getFWDCodeHandle() → 跳转微信 OAuth (snsapi_userinfo)
    ↓
微信回调: 原 URL + &code=xxx&isBack=true
    ↓
initHandle() 再次执行:
    1. isBack = true → 提取 code
    2. 有 code → 继续加载
    ↓
调 propagationChainHandle()
    → trackingMaterial({
        materialId: 活动ID,
        materialType: '1',
        userId: '',                          ← 客户首次为空
        code: 微信OAuth的code,               ← 关键！用 code 换微信用户信息
        chain: 从URL取,
        senderId: 从URL取(代理人ID),
        directAgentId: 从URL取(代理人ID),
        parentPropagationId: 从URL取,
        shareTime: 从URL取
    })
    ↓
后端处理:
    1. userId 为空 → 用 code 换微信信息 (openId, unionId, nickname)
    2. unionId 查 blade_user → 查不到 → isReaderAgent = false → 是客户
    3. setCustomerType(2)    // 客户
    4. 查 directAgentId 对应的代理人 → 设置名片信息 (agentCard)
    5. handleCustomerInfo() → 用 openId + directAgentId 查/创建 client_user 记录（获客）
    6. handleAgentShareBehavior() → 记录代理人分享行为 → 写入 customer_client_user_behavior (type=1, behavior=1)
    7. handleClientBrowsingBehavior() → 记录客户浏览行为 → 写入 customer_client_user_behavior (type=1, behavior=2)
    8. sendViewMessage() → 发送互动消息给代理人（flag=view_article, 含 activityId+activityName+posterUrl）
    ↓
返回: {
    userId: 客户ID (client_user.id),
    customerType: 2,
    agentCard: {...},
    browsingId: xxx,
    chain: xxx,
    directAgentId: 代理人ID,
    ...
}
    ↓
前端:
    localStorage.setItem('USERID', 客户ID)
    customerType = 2 → isAgent = false
    ↓
加载活动详情 getActivityData()
    → 调 /h5/activity/{id}
    → 调 /h5/activity/{id}/status（用 userId 查参与状态）
```

### 2.3 客户报名活动

```
客户点击"立即报名" → 跳转 /activity/register
    → URL 带 activityId + userId + directAgentId
    ↓
填写表单 → 提交
    → POST /h5/activity/{id}/register?userId=xxx&directAgentId=xxx
    → body: [{name: "张三"}, {phone: "138xxx"}, ...]
    ↓
后端 UserActivityServiceImpl.registerActivity():
    1. 校验活动存在、状态、名额、重复报名
    2. 从 formData 提取 userName/userPhone
    3. 获取头像：blade_user → client_user → 前端传入 userAvatarParam
    4. 保存/更新 surge_user_activity 记录（status=1, checkinStatus=0）
    5. incrementRegistrationCount()
    6. 发送消息（仅 userId != receiverAgentId 时发，自己报名不通知自己）：
       → sendActivityMessage(activity_register, receiverAgentId)
    7. 保存行为记录：
       → saveBehavior(activityId, userId, teamUserId=receiverAgentId, behavior=3)
       → 通过 propagationChain 表反查 chainId 关联
```

### 2.4 客户签到

```
客户点击"签到"
    → POST /h5/activity/{id}/checkin?userId=xxx&directAgentId=xxx
    ↓
后端 UserActivityServiceImpl.checkinActivity():
    1. 校验活动存在、状态、是否已签到
    2. 仅签到模式自动创建记录
    3. 更新 checkinStatus=1, checkinTime
    4. incrementCheckinCount()
    5. 发送消息（自己签到不通知自己）：
       → sendActivityMessage(activity_checkin, receiverAgentId)
    6. 保存行为记录：
       → saveBehavior(activityId, userId, teamUserId, behavior=4)
```

### 2.5 用户身份判断表

| 场景 | userId | code | 后端判断 | customerType |
|------|--------|------|---------|-------------|
| App 代理人分享 | 有(代理人ID) | 空 | userId 查 blade_user → 找到 | 1 (代理人) |
| 微信客户首次打开 | 空 | 有(OAuth code) | code 换微信信息 → unionId 查 blade_user → 查不到 | 2 (客户) |
| 微信客户再次打开 | 有(localStorage) | 空 | userId 查 blade_user → 查不到 → 查 client_user → 找到 | 2 (客户) |
| 代理人在微信打开自己分享的链接 | 有(代理人ID) | 空 | userId 查 blade_user → 找到 | 1 (代理人) |

## 三、数据存储表

### 3.1 传播链表 `intrument_propagation_chain`

| 字段 | 说明 |
|------|------|
| id | 主键（chainId） |
| chain | 链路标识 |
| material_id | 活动ID |
| material_type | 1=活动 |
| sender_id | 发送者ID |
| receiver_id | 接收者ID |
| direct_agent_id | 直接代理人ID |
| parent_id | 父传播链ID |

### 3.2 行为记录表 `customer_client_user_behavior`

| 字段 | 说明 |
|------|------|
| type | 0=文章, 1=活动, 2=评测, 3=名片, 4=测评 |
| behavior | 1=转发, 2=浏览, 3=活动报名, 4=活动签到 |
| source_id | 活动ID |
| team_user_id | 代理人ID |
| client_user_id | 客户ID |
| chain_id | 关联传播链ID |
| share_time | 分享时间戳 |

### 3.3 消息表 `blade_message`

| Flag | 触发场景 | 消息来源 |
|------|---------|---------|
| `view_article` | 客户浏览活动 | blade-instrument 传播链 |
| `share_article` | 客户转发活动 | blade-instrument 传播链 |
| `activity_register` | 客户报名活动 | surge-mvp 报名接口 |
| `activity_checkin` | 客户签到活动 | surge-mvp 签到接口 |

**消息内容 JSON:**
```json
{
    "activityId": "活动ID",
    "activityName": "活动名称",
    "posterUrl": "封面图URL",
    "articleImage": "封面图URL（兼容）",
    "renderName": "客户昵称",
    "renderAvatar": "客户头像",
    "renderId": "客户ID",
    "senderId": "发送者ID",
    "teamUserId": "代理人ID",
    "chainId": "传播链ID",
    "title": "活动名称"
}
```

## 四、blade-instrument 改动点

在 `ConsultationArticleServiceImpl` 中，涉及 materialType 判断的位置都加了 `"1".equals(materialType)` 的活动分支:

1. `checkMaterialExists` — 查 `surge_activity` 表获取活动名称
2. `fillActivityInfo` — 新增方法，填充活动名称+封面到消息 JSON
3. 代理人分享行为消息 (~1507行) — `else if ("1".equals(materialType))` + fillActivityInfo
4. 客户浏览行为消息 (~1805行) — 同上
5. 客户转发消息 (sendShareMessage ~2577行) — 同上
6. 客户浏览消息 (sendViewMessage ~2625行) — 同上
7. `getBehaviorInfo` SQL — UNION 加了 surge_activity
8. `getCustomerBehaviorInfo` SQL — UNION 加了 surge_activity
9. `behaviorList` SQL — UNION 加了 surge_activity

**使用 `SurgeActivityMapper`（同库直查）:**
```java
@Mapper
public interface SurgeActivityMapper {
    @Select("SELECT name FROM surge_activity WHERE id = #{id}")
    String getActivityNameById(@Param("id") String id);

    @Select("SELECT poster_url FROM surge_activity WHERE id = #{id}")
    String getActivityPosterById(@Param("id") String id);
}
```

## 五、surge-mvp 改动点

1. `UserActivityServiceImpl.saveBehavior()` — 报名/签到后写入 `customer_client_user_behavior`（type=1, behavior=3/4），通过 propagationChain 反查 chainId
2. `UserActivityServiceImpl.sendActivityMessage()` — 报名/签到后发消息给 directAgentId（不通知自己）
3. `AppActivityController` — 参与者列表返回前补充 client_user/blade_user 头像名称
4. `H5ActivityController` — 报名/签到接口加 directAgentId 参数

## 七、与问卷传播链对比

| 对比项 | 活动 (type=1) | 问卷 (type=5) |
|--------|-------------|-------------|
| **MaterialConstant** | MATERIAL_TYPE_ACTIVITY = 1 | MATERIAL_TYPE_QUESTIONNAIRE = 5 |
| **checkMaterialExists** | surgeActivityMapper.getActivityNameById | surgeQuestionnaireMapper.getTitleById |
| **fillInfo** | fillActivityInfo (name + poster_url) | fillQuestionnaireInfo (title + cover_image) |
| **消息 JSON** | activityId + activityName + posterUrl | questionnaireId + questionnaireName + posterUrl |
| **参与行为** | behavior=3(报名) + behavior=4(签到) | questionnaire_submit |
| **参与消息** | activity_register + activity_checkin | questionnaire_submit |
| **自己参与不通知** | ✅ userId != receiverAgentId | ✅ 仅客户提交时发 |
| **前端识别** | isActivityMsg (content.activityId) | isQuestionnaireMsg (content.questionnaireId) |
