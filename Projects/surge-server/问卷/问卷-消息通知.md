# 消息通知

## 传播链消息 (blade-instrument)

`trackingMaterial` 接口在客户浏览/分享时自动记录：

| flag | 触发时机 |
|------|----------|
| view_article | 客户浏览了代理人分享的问卷 |
| share_article | 客户转发了问卷 |

### 消息内容JSON

```json
{
    "questionnaireId": "问卷ID",
    "questionnaireName": "问卷标题",
    "posterUrl": "封面图URL",
    "articleImage": "封面图URL",
    "renderName": "客户昵称",
    "renderAvatar": "客户头像",
    "renderId": "客户ID",
    "senderId": "发送者ID",
    "teamUserId": "代理人ID",
    "chainId": "传播链ID",
    "title": "问卷标题"
}
```

## 问卷参与消息 (surge-mvp)

| flag | 触发时机 | 接收人 |
|------|----------|--------|
| questionnaire_submit | 客户提交问卷 | directAgentId(代理人) |

> [!note] 仅客户提交时发送
> App用户自己填写不发消息。
> 需要请求体带 `directAgentId`（从分享链接URL参数获取）。

## App端展示

### 消息中心首页 (msgItem.vue)

通过 `aroundText` computed 生成文案：
- `questionnaireId` 存在 → "浏览了您分享的调查问卷"
- `activityId` 存在 → "浏览了您分享的活动"
- 都没有 → "浏览了您转发的文章"

### 互动消息列表 (interactionMsgItem.vue)

- 问卷消息卡片：展示封面图 + 问卷标题
- 点击跳转 `surveyDetail` 详情页
- `questionnaire_submit` flag → "参与了您分享的调查问卷"

questionnaire_submit("问卷参与", "互动消息")

## blade-instrument改动点

- `SurgeQuestionnaireMapper`：同库直查 surge_questionnaire 表
- `fillQuestionnaireInfo()`：填充问卷标题+封面到消息JSON
- `checkMaterialExists()`：问卷类型查表获取真实标题
- 4处消息构建都加了 materialType=5 的分支
