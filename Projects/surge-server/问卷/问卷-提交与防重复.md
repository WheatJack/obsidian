# 提交与防重复

## App用户填写

```
App surveyDetail → 点"立即填写" → surveyFill原生页面
    ↓
填写完成 → 调 /wechat/questionnaire/submit
    请求头带Blade-Auth token → 后端SecureUtil.getUser()拿到userId
    ↓
uni.$emit('surveySubmitted') → navigateBack
    → 详情页弹成功浮层3秒 → 按钮变"已填写"灰色
```

## 微信客户填写

```
微信打开问卷 → 填写 → 调 /wechat/questionnaire/submit
    请求体: { questionnaireId, openId, userId(USERID),
             nickname, avatar, directAgentId, answerList }
    ↓
后端:
    1. 从req.userId或SecureUtil取userId
    2. 防重复校验(checkSubmitted)
    3. 存入submission(openId + userId)
    4. 客户(userId为null) + 有directAgentId → 发互动消息
    ↓
跳转 /survey/result → 感谢语 + 3秒倒计时
```

## 防重复提交

### 后端

- `findSubmission()`：按openId或userId任一匹配查submission
- `submitAnswer()` 开头调 `checkSubmitted()`，重复提交抛异常

### App端

- 详情页loadData时调check接口 → hasSubmitted → 按钮灰色

### Wechat端

- initHandle → checkAlreadySubmitted → 回填已提交答案 + 只读模式
- 不跳filled页，留在问卷页展示绿色banner "已提交过此问卷"
- 提交按钮置灰显示"已提交"

> [\!tip] check接口返回答案
> check接口在 `submitted=true` 时同时返回 `answers` 数组。
> 前端回填时注意将选项ID统一转字符串：
> `a.answerOptions.map(o => String(o))`
