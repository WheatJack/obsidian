# 传播链流程

问卷复用文章/活动的 `trackingMaterial` 接口，`materialType = '5'`。

## 第一步：App代理人分享

```
代理人在App点击"分享给客户"
    ↓
App调 trackingMaterial({
    materialId: 问卷ID,
    materialType: '5',
    userId: 代理人ID,
    code: '',
    chain: '', senderId: '', directAgentId: ''
})
    ↓
后端处理:
    1. userId有值 → 查blade_user → 找到 → customerType=1(代理人)
    2. 生成唯一chain、browsingId、shareTime
    3. 记录代理人浏览行为
    ↓
返回: { userId, senderId=代理人ID, directAgentId=代理人ID, chain, shareTime }
    ↓
App拼装分享链接并分享到微信
```

## 第二步：微信客户打开

```
客户在微信点开链接
    ↓
initHandle():
    1. 检查localStorage.USERID → 没有
    2. getFWDCodeHandle() → 微信OAuth(snsapi_userinfo)
    ↓
微信回调: 原URL + &code=xxx&isBack=true
    ↓
调 trackingMaterial({
    materialId: 问卷ID,
    materialType: '5',
    userId: '',
    code: 微信OAuth的code,  ← 关键！用code换微信用户信息
    chain/senderId/directAgentId: 从URL取
})
    ↓
后端处理:
    1. userId为空 → 用code换微信信息(openId, unionId)
    2. unionId查blade_user → 查不到 → customerType=2(客户)
    3. 查/创建client_user记录
    4. 记录客户浏览行为
    ↓
前端:
    localStorage.setItem('USERID', 客户ID)
    customerType=2 → isUser=false
```

## 身份判断表

| 场景 | userId | code | customerType |
|------|--------|------|-------------|
| App代理人分享 | 有(代理人ID) | 空 | 1 (代理人) |
| 微信客户首次打开 | 空 | 有(OAuth code) | 2 (客户) |
| 微信客户再次打开 | 有(localStorage) | 空 | 2 (客户) |

> [!warning] 字段名
> 后端返回的是 `customerType`（1=代理人，2=客户）。
> 不是 `readerType`。前端判断时用 `resData.customerType`。
