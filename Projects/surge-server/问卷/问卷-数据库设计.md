# 数据库设计

## 表结构

| 表名 | 说明 |
|------|------|
| surge_questionnaire | 问卷主表 |
| surge_questionnaire_question | 题目表 |
| surge_questionnaire_option | 选项表 |
| surge_questionnaire_submission | 提交记录表 |
| surge_questionnaire_answer | 答题明细表 |

## 关键字段

### surge_questionnaire

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 雪花ID |
| title | VARCHAR(255) | 问卷标题 |
| scope | TINYINT | 活动范围: 1=仅客户 2=客户和用户 3=仅用户 |
| target_tenants | VARCHAR(2000) | 目标租户ID JSON数组 |
| auth_type | TINYINT | 固定为1(微信授权) |
| tenant_id | VARCHAR(12) | 租户隔离 |
| status | TINYINT | 0=草稿 1=已上架 2=已下架 3=已结束 |
| submit_count | INT | 提交数量(原子更新) |
| visit_count | INT | 访问数量(原子更新) |

### surge_questionnaire_submission

| 字段 | 类型 | 说明 |
|------|------|------|
| user_id | BIGINT | App用户ID(blade_user.id) |
| open_id | VARCHAR(64) | 微信openId(客户) |
| nickname | VARCHAR(100) | 昵称 |
| avatar | VARCHAR(500) | 头像 |
| duration | INT | 填写耗时(秒) |

> [!note] 计数器更新
> `submit_count` 和 `visit_count` 使用原子 SQL 更新:
> `UPDATE ... SET submit_count = IFNULL(submit_count, 0) + 1`
> 避免并发丢数据。
