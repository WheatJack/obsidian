# 问卷调查模块

> 同步自: `surge-server/surge-mvp/doc/questionnaire-module.md`
> 项目: [[surge-server]]

## 一、模块概述

问卷调查模块支持在后台 Admin 创建问卷，通过 App 分享给微信客户填写，或 App 用户直接填写。涉及三端：

| 端 | 项目 | 说明 |
|---|------|------|
| Admin 后台 | surge-admin | 问卷创建、编辑、上下架、统计 |
| App 端 | surge-app | 问卷列表、详情、填写、分享 |
| 微信 H5 端 | surge-wechat | 客户填写问卷 |
| 后端服务 | surge-mvp | 问卷 CRUD、提交、统计接口 |
| 传播链服务 | blade-instrument | 传播链追踪、消息通知 |

## 二、数据库表

| 表名 | 说明 |
|------|------|
| surge_questionnaire | 问卷主表 |
| surge_questionnaire_question | 题目表 |
| surge_questionnaire_option | 选项表 |
| surge_questionnaire_submission | 提交记录表（含 user_id、open_id） |
| surge_questionnaire_answer | 答题明细表 |

### 关键字段

**surge_questionnaire**
- `scope`: 活动范围 1=仅客户 2=客户和用户 3=仅用户
- `target_tenants`: 目标租户 JSON 数组，scope=2/3 时由后端自动填入当前 tenantId
- `auth_type`: 固定为 1（微信授权）
- `tenant_id`: 租户隔离

**surge_questionnaire_submission**
- `user_id`: App 用户 ID（blade_user.id），App 端用户填写时由后端 SecureUtil 获取
- `open_id`: 微信 openId，微信客户填写时由传播链获取

## 四、传播链流程

问卷的传播链复用文章/活动的 `trackingMaterial` 接口，`materialType = '5'`。

### 4.1 App 代理人分享问卷到微信

```
代理人在 App 点击"分享给客户"
    ↓
App 调 trackingMaterial({
    materialId: 问卷ID,
    materialType: '5',        ← 问卷类型
    userId: 代理人ID,          ← 从 storage 取
    code: '',                  ← App 端为空
    chain: '',                 ← 首次分享为空
    senderId: '',
    directAgentId: '',
    parentPropagationId: '',
    shareTime: ''
})
    ↓
后端处理:
    1. userId 有值 → 查 blade_user → 找到 → isReaderAgent = true
    2. setCustomerType(1)    // 代理人
    3. 生成唯一 chain、browsingId、shareTime
    4. 记录代理人浏览行为
    5. fillQuestionnaireInfo() 填充问卷标题+封面
    ↓
返回: {
    userId: 代理人ID,
    senderId: 代理人ID,
    directAgentId: 代理人ID,
    chain: 唯一链ID,
    shareTime: 时间戳,
    browsingId: 浏览记录ID,
    customerType: 1
}
    ↓
App 拼装分享链接:
    https://surge-wechat.insbean.com/survey
        ?id=问卷ID
        &browsingId=xxx
        &chain=xxx
        &directAgentId=代理人ID
        &senderId=代理人ID
        &shareTime=xxx
    ↓
分享到微信
```

### 4.2 微信客户打开问卷链接

```
客户在微信点开链接
    ↓
initHandle():
    1. 检查 localStorage.USERID → 没有
    2. 检查 userStore.wxUserInfo → 没有
    3. 调 getFWDCodeHandle() → 跳转微信 OAuth (snsapi_userinfo)
    ↓
微信回调: 原 URL + &code=xxx&isBack=true
    ↓
initHandle() 再次执行:
    1. isBack = true → 提取 code
    2. 有 code → 继续加载
    ↓
加载问卷详情 getSurveyDetail(id)
    ↓
调 trackingMaterial({
    materialId: 问卷ID,
    materialType: '5',
    userId: '',                          ← 客户首次为空
    code: 微信OAuth的code,               ← 关键！用 code 换微信用户信息
    chain: 从URL取,
    senderId: 从URL取(代理人ID),
    directAgentId: 从URL取(代理人ID),
    parentPropagationId: 从URL取,
    shareTime: 从URL取
})
    ↓
后端处理:
    1. userId 为空 → 用 code 换微信信息 (openId, unionId, nickname)
    2. unionId 查 blade_user → 查不到 → isReaderAgent = false → 是客户
    3. setCustomerType(2)    // 客户
    4. 查 directAgentId 对应的代理人 → 设置名片信息
    5. 用 openId + directAgentId 查/创建 client_user 记录
    6. 记录客户浏览行为、代理人分享行为
    7. 发送互动消息给代理人（view_article）
    ↓
返回: {
    userId: 客户ID,
    customerType: 2,
    agentCard: {...},
    browsingId: xxx,
    chain: xxx,
    directAgentId: 代理人ID,
    ...
}
    ↓
前端:
    localStorage.setItem('USERID', 客户ID)
    customerType = 2 → isUser = false
    ↓
根据 scope 判断权限:
    scope=1(仅客户) → 客户可填 ✓
    scope=2(客户和用户) → 客户可填 ✓
    scope=3(仅用户) → 客户不可填 → 跳转 /survey/restricted
    ↓
检查是否已提交 → check 接口
    已提交 → 回填答案，提交按钮置灰
    未提交 → 正常填写
```

### 4.3 用户身份判断表

| 场景 | userId | code | 后端判断 | customerType |
|------|--------|------|---------|-------------|
| App 代理人分享 | 有(代理人ID) | 空 | userId 查 blade_user → 找到 | 1 (代理人) |
| 微信客户首次打开 | 空 | 有(OAuth code) | code 换微信信息 → unionId 查 blade_user → 查不到 | 2 (客户) |
| 微信客户再次打开 | 有(localStorage) | 空 | userId 查 blade_user → 查不到 → 查 client_user → 找到 | 2 (客户) |
| 代理人在微信打开自己分享的链接 | 有(代理人ID) | 空 | userId 查 blade_user → 找到 | 1 (代理人) |

## 五、提交流程

### 5.1 App 用户填写

```
App surveyDetail → 点"立即填写" → surveyFill 原生页面
    ↓
填写完成 → 调 /wechat/questionnaire/submit
    请求头带 Blade-Auth token
    → 后端 SecureUtil.getUser() 拿到 userId
    → 存入 submission.user_id
    ↓
uni.$emit('surveySubmitted') → navigateBack
    → 详情页弹成功浮层 3 秒 → 按钮变"已填写"灰色
```

### 5.2 微信客户填写

```
微信打开问卷 → 填写 → 调 /wechat/questionnaire/submit
    请求体: {
        questionnaireId, openId, userId(USERID), nickname, avatar,
        directAgentId(从URL取), answerList
    }
    → 后端:
        1. 从 req.userId 或 SecureUtil 取 userId
        2. 防重复提交校验 (checkSubmitted)
        3. 存入 submission (openId + userId)
        4. 如果是客户(userId为null) + 有 directAgentId → 发互动消息给代理人
    ↓
跳转 /survey/result → 显示感谢语 + 3秒倒计时
```

### 5.3 防重复提交

后端 `checkSubmitted` 方法:
- 按 openId 或 userId 任一匹配 → 已提交
- `submitAnswer` 方法开头调 `checkSubmitted`，重复提交直接抛异常

前端双重校验:
- App: 详情页 loadData 时调 check 接口 → hasSubmitted → 按钮灰色
- Wechat: initHandle → checkAlreadySubmitted → 回填已提交答案 + 只读模式

## 六、消息通知

### 6.1 传播链消息（blade-instrument）

`trackingMaterial` 接口在客户浏览/分享时自动记录:
- `view_article` (flag): 客户浏览了代理人分享的问卷
- `share_article` (flag): 客户转发了问卷

消息内容 JSON:
```json
{
    "questionnaireId": "问卷ID",
    "questionnaireName": "问卷标题",
    "posterUrl": "封面图URL",
    "articleImage": "封面图URL",
    "renderName": "客户昵称",
    "renderAvatar": "客户头像",
    "renderId": "客户ID",
    "senderId": "发送者ID",
    "teamUserId": "代理人ID",
    "chainId": "传播链ID",
    "title": "问卷标题"
}
```

### 6.2 问卷参与消息（surge-mvp）

客户提交问卷后，发送 `questionnaire_submit` 消息给 directAgentId:
- 仅客户提交时发送（App 用户自己填不发）
- 需要请求体带 `directAgentId`

### 6.3 MessageEnum

```java
questionnaire_submit("问卷参与", "互动消息")
```

### 6.4 MaterialConstant

```java
MATERIAL_TYPE_QUESTIONNAIRE = 5
```

## 七、blade-instrument 改动点

在 `ConsultationArticleServiceImpl` 中，涉及 materialType 判断的 4 处位置都需要加上 `"5".equals(materialType)` 的问卷分支:

1. `checkMaterialExists` — 查 surge_questionnaire 表获取标题
2. `fillQuestionnaireInfo` — 填充问卷标题+封面到消息 JSON
3. 代理人分享行为消息 (~1514行)
4. 客户分享行为消息 (~1812行)
5. 客户转发消息 (~2636行)
6. 客户浏览消息 (~2686行)

使用 `SurgeQuestionnaireMapper`（同库直查）:
```java
@Mapper
public interface SurgeQuestionnaireMapper {
    @Select("SELECT title FROM surge_questionnaire WHERE id = #{id} AND is_deleted = 0")
    String getTitleById(@Param("id") String id);

    @Select("SELECT cover_image FROM surge_questionnaire WHERE id = #{id} AND is_deleted = 0")
    String getCoverImageById(@Param("id") String id);
}
```

## 八、Scope（活动范围）逻辑

| scope | 说明 | App 用户 | 微信客户 |
|-------|------|---------|---------|
| 1 | 仅客户 | 只能分享，不能填写 | 可以填写 |
| 2 | 客户和用户 | 可分享+可填写 | 可以填写 |
| 3 | 仅用户 | 可填写 | 不可访问(restricted) |

- `target_tenants`: 保存时后端自动用 `SecureUtil.getTenantId()` 填入
- App 列表查询: 按 `tenant_id` 精确匹配，不需要前端传 tenantId
