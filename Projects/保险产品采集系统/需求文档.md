# 财产保险产品数据采集系统 - 需求文档

## 1. 项目背景

中国保险行业协会提供了一个**财产保险公司自主注册产品查询系统**（`https://cxcx.iachina.cn`），包含了全国所有财产保险公司注册的保险产品信息，包括产品条款、费率等关键数据。

为了将这些产品数据纳入我们的系统进行管理和分析，需要建设一套自动化的数据采集系统。

## 3. 功能需求

### 3.1 爬虫脚本（Python）

- **F1**: 支持按产品类别、产品状态分页爬取所有产品列表
- **F2**: 支持获取每个产品的详情页信息（注册时间、停售时间、客服电话）
- **F3**: 支持下载产品条款和费率文件内容
- **F4**: 爬取前先调用 MVP 接口判断数据是否已存在，存在则跳过
- **F5**: 爬取后调用 MVP 接口保存数据
- **F6**: 支持命令行参数传入 cookies
- **F7**: 每分钟检测 session 有效性
- **F8**: session 失效时自动停止并记录日志

### 3.2 MVP 服务接口（Java）

#### 产品数据接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 判断存在 | GET | `/insurance/product/exists?regNo=xxx` | 根据注册号判断 |
| 保存/更新 | POST | `/insurance/product/saveOrUpdate` | 根据注册号去重 |
| 批量保存 | POST | `/insurance/product/batchSave` | 批量保存 |
| 分页查询 | GET | `/insurance/product/page` | 支持公司/产品/类别/状态筛选 |
| 按注册号查询 | GET | `/insurance/product/getByRegNo` | |

#### 爬虫管理接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 更新Cookies | POST | `/insurance/crawler/updateCookies` | 手动更新cookies |
| 获取Cookies | GET | `/insurance/crawler/getCookies` | |
| 触发爬虫 | POST | `/insurance/crawler/start` | 可选传cookies |
| 查询状态 | GET | `/insurance/crawler/status` | 运行状态 |
| 测试邮件 | POST | `/insurance/crawler/testMail` | 测试邮件配置 |

### 3.3 定时任务

- **每天凌晨2点**: 自动触发爬虫采集
- **每分钟**: 检测 session 有效性

### 3.4 邮件通知

| 场景 | 触发条件 | 内容 |
|------|----------|------|
| 采集完成 | 爬虫正常结束 | 新增/跳过/失败数量、耗时 |
| Session失效 | 检测到session过期 | 提示更新cookies（每4小时最多1次） |
| 任务异常 | 爬虫进程异常退出 | 错误信息 |

## 5. 约束与限制

1. **认证方式**: 网站使用滑动验证码，无法完全自动化登录，需要人工首次登录获取cookies
2. **Session时效**: cookies 有过期时间，过期后需重新登录
3. **请求频率**: 爬取间隔 0.3 秒/次，避免被封
4. **数据量预估**: 19个类别 × 2种状态，预计总数据量 1-2 万条
