# 财产保险产品数据采集系统 - 部署文档

## 1. 系统架构

```
┌──────────────────────────────────────────────────────────┐
│                        服务器                             │
│                                                          │
│  宿主机                          Docker                   │
│  ┌──────────────────┐           ┌─────────────────────┐  │
│  │ /opt/crawler/     │──HTTP──▶ │ MVP 服务 (:8088)     │  │
│  │ cxcx_crawler.py   │◀─HTTP── │                       │  │
│  │                   │          │ ├ 产品数据 CRUD       │  │
│  │ cron 每天凌晨2点   │          │ ├ Cookies 管理       │  │
│  └────────┬──────────┘          │ ├ 邮件通知            │  │
│           │                     └──────────┬────────────┘  │
│      爬取数据                              │               │
│           │                     ┌──────────▼────────────┐  │
│           ▼                     │ MySQL                  │  │
│  cxcx.iachina.cn               │ └ insurance_product    │  │
│  (目标网站)                     └───────────────────────┘  │
│                                                          │
│  完成/失效/异常 ──邮件──▶ 开发 & 运营                      │
└──────────────────────────────────────────────────────────┘
```

**关键点: Python 爬虫跑在宿主机上，MVP 服务跑在 Docker 里，两者只通过 HTTP 通信。**

## 2. 环境要求

| 项目 | 宿主机 | Docker |
|------|--------|--------|
| Python 3.8+ | 必须 | 不需要 |
| Java 17 | 不需要 | MVP 镜像自带 |
| pip: requests | 必须 | 不需要 |
| MySQL 5.7+ | - | 已有 |

## 3. 部署步骤

### 3.1 数据库初始化

```sql
-- 只需一张表
CREATE TABLE `insurance_product` (
  `id` varchar(64) NOT NULL COMMENT '主键ID',

```

### 3.2 部署 MVP 服务 (Docker)

正常跟随 surge-mvp 项目部署即可，新增模块不需要额外配置。

### 3.3 部署 Python 爬虫 (宿主机)

```bash
# 1. 创建目录
mkdir -p /opt/crawler

# 2. 上传脚本
# 源文件: surge-mvp/src/main/java/com/surge/insurance/crawler/cxcx_crawler.py
scp cxcx_crawler.py server:/opt/crawler/cxcx_crawler.py

# 3. 安装依赖
pip3 install requests

# 4. 验证
python3 /opt/crawler/cxcx_crawler.py --help
```

### 3.4 配置定时任务 (宿主机 cron)

```bash
crontab -e
```

添加以下内容：

```cron
# 每天凌晨2点执行保险产品采集（从 MVP 接口获取 cookies）
0 2 * * * /usr/bin/python3 /opt/crawler/cxcx_crawler.py --mvp-base http://localhost:8088 >> /opt/crawler/cron.log 2>&1
```

> 注意 `--mvp-base` 的地址要改成 MVP Docker 容器实际可访问的地址（如 `http://host.docker.internal:8088` 或宿主机 IP）。

### 3.5 配置邮件收件人

修改 `MailNotifyService.java` 中的 `NOTIFY_RECIPIENTS`：

```java
private static final List<String> NOTIFY_RECIPIENTS = List.of(
    "dev@yourcompany.com",
    "ops@yourcompany.com"
);
```

## 4. 使用方式

### 4.1 首次使用（3步）

**Step 1: 获取 Cookies**

浏览器登录 `https://cxcx.iachina.cn`，完成滑动验证后，F12 → Application → Cookies → 复制所有 cookie。

**Step 2: 传入 Cookies**

```bash
curl -X POST http://localhost:8088/insurance/crawler/updateCookies \
  -H "Content-Type: application/json" \
  -d '{"cookies": "JSESSIONID=13tn8pRuJCMa-uebomd7p5pR2rkLrNKtv6qW8GxdzuaXI4RArtKX!1311178019;lasttime=45ff6a6c-3764-4f05-8243-508037b67451"}'
```

**Step 3: 手动触发一次（或等 cron 自动执行）**

```bash
# 在宿主机上直接运行
python3 /opt/crawler/cxcx_crawler.py --mvp-base http://localhost:8088

# 或者指定 cookies 直接跑（不依赖 MVP 存的 cookies）
python3 /opt/crawler/cxcx_crawler.py --cookies "JSESSIONID=xxx" --mvp-base http://MVP地址:8088
```

### 4.2 日常运维

| 收到什么邮件 | 做什么 |
|-------------|--------|
| 采集完成 | 无需操作，确认数据量即可 |
| Session 失效 | 重新登录网站 → 获取 cookies → 调用 updateCookies 接口 |
| 任务异常 | 检查 `/opt/crawler/cxcx_*.log` 日志 |

### 4.3 查询状态

```bash
# 查看爬虫状态
curl http://MVP地址:8088/insurance/crawler/status

# 查看产品数据
curl "http://MVP地址:8088/insurance/product/page?page=1&size=10"

# 测试邮件是否正常
curl -X POST http://MVP地址:8088/insurance/crawler/testMail
```

## 5. 接口速查

### 人工操作

| 操作 | 方法 | 路径 |
|------|------|------|
| 传入Cookies | POST | `/insurance/crawler/updateCookies` |
| 查看状态 | GET | `/insurance/crawler/status` |
| 测试邮件 | POST | `/insurance/crawler/testMail` |
| 查询产品 | GET | `/insurance/product/page` |

### Python 爬虫回调（自动调用）

| 操作 | 方法 | 路径 |
|------|------|------|
| 获取Cookies | GET | `/insurance/crawler/getCookies` |
| 判断产品存在 | GET | `/insurance/product/exists?regNo=xxx` |
| 保存产品 | POST | `/insurance/product/saveOrUpdate` |
| 完成通知 | POST | `/insurance/crawler/notifyComplete` |
| 失效通知 | POST | `/insurance/crawler/notifySessionExpired` |
| 异常通知 | POST | `/insurance/crawler/notifyError` |

## 7. 注意事项

1. **Cookies 存在 MVP 服务内存中**，MVP 服务重启后需重新调用 `updateCookies` 传入
2. **Python 脚本必须放在** `/opt/crawler/cxcx_crawler.py`，日志也写在该目录
3. **cron 中的 MVP 地址**要确保宿主机能访问到 Docker 里的服务
4. **首次全量采集**数据量大（1-2万条），耗时较长，建议先用 `--category 309` 测试单个类别
